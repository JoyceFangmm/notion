介绍

LLM（Large Language Model，大语言模型）：核心引擎，把文字拆成 token，然后通过计算预测一个token后面最该接哪一个token

Token：大模型理解文字的最小单元，每个 token 对应一个数字 ID，叫Token ID

Prompt（提示词）：你对模型说的话

Prompt Engineering（提示词工程）：把话说清楚的方法论，问得越好答得越好

Context（上下文）：每次发消息时系统把历史对话打包发给大模型

Memory（记忆）：上下文窗口有限，大模型把历史对话压缩总结关键信息保存

RAG（检索增强生成）：让大模型能读你的私有资料，切片存知识库，提问时检索相关片段拼进上下文再生成回复。
例如：AI客服，AI助手，都是在用RAG技术。

Function Calling（函数调用）：让大模型能调用外部工具，大模型输出结构化调用指令，程序执行后把结果还给大模型

MCP（Model Context Portocol，模型上下文协议）：统一所有第三方工具接口的协议。

MCP不是 Function Calling 的替代品：
Function Calling 是「模型怎么输出指令，让模型按照约定格式输出调用指令」
MCP 是「工具怎么被统一接入，让所有工具都遵循同一种格式被模型接入」

Agent（智能体）：接到任务后，会自己思考该怎么做，自己规划步骤，自己调用外部工具，自己记录每一步的结果。Agent就是在大模型的基础上，把这些能力打包成一个能独立干活的系统。

Skill（技能）：把所有的偏好和规则写进一个skill内，下次Agent接到类似任务时，会参考这份文件，按规定办事。是可以重复使用的。

Prompt与Skill的区别：
Prompt是一次性话术，零散的表达；像便签。
Skill是可复用的能力，结构化的能力单元；是可重复使用的程序化能力；像书。

Harness Engineering（驾驭工程）：首先将最全面的上下文提供给模型，避免其失忆；同时给Agent划定边界；最后能自动验收任务成果，第一时间给出反馈，并且引导修复。
核心：构造一个AI友好工作环境，确保在可控的范围内爆发生产力。

总结

简单记忆

LLM：大脑
Token：大脑能读的文字颗粒
Prompt：你下达的指令
Prompt Engineering：把指令写清楚的方法
Context：大脑当前能看到的材料
Memory：大脑长期记住的信息
RAG：先查资料再回答
Function Calling：调用具体函数/API
MCP：统一连接外部工具的协议
Skill：封装好的专项能力
Harness：把模型、工具、流程跑起来的框架
Agent：能自己规划并执行任务的智能体
